# 프로젝트 계획서

# 1. 프로젝트 개요

- 프로젝트 제목: 따뜻한하루 - 문자로 가족이 되는 AI 케어 서비스
- 학번: 3401
- 이름: 고명진

### 1-1 프로젝트 한 줄 설명

- 설명 : 문자 한 통으로 어르신의 외로움을 해소하고 건강·안전을 지키는 가족 같은 AI 케어 서비스

---

# 2. 문제 정의 (Problem Definition)

## 2-1 해결하려는 문제

- 해결하는 문제: 독거노인의 정서적 외로움, 건강 관리 미흡, 가족과의 단절
- 문제 발생 배경:
    - 한국 65세 이상 독거노인 300만 명 돌파(2025년 기준)
    - 스마트폰 보급률이 높아도 디지털 소외가 심각함
    - 기존 서비스는 AI 전화·로봇 중심으로 접근성이 낮음
- 왜 중요한 문제인가: 고독사의 주요 원인 중 하나가 외로움이며, 예방적 정서 케어가 부족함

---

## 2-2 프로젝트 목표

- 목표 : AI 문자 기반으로 어르신의 일상 대화를 제공하여 외로움 지수 30% 감소를 목표로 하고, 가족 모니터링을 통해 건강 이상을 조기 발견하는 시스템을 구축한다.

---

# 3. 사용자 분석 (Target User Analysis)

- 대상 사용자:
    - 어르신(Primary): 70대 이상 독거노인, 문자 사용이 가능한 스마트폰 사용자
    - 가족(Secondary): 30~50대 자녀, 부모 건강 상태를 비침투적으로 모니터링할 필요가 있는 사용자
- 사용 환경:
    - 어르신: 피처폰 또는 저사양 스마트폰, 문자 중심 사용
    - 가족: PC/스마트폰 웹 브라우저(웹 또는 PWA)
- 사용자가 겪는 문제:
    - 어르신: 외로움, 약 복용 망각, 응답 지연 시 가족 불안 증가
    - 가족: 부모 상태 확인 어려움, 잦은 통화 부담
- 사용 목적:
    - 어르신: 따뜻한 일상 대화, 건강 리마인더
    - 가족: 실시간 안심 모니터링, 부담 없는 상태 확인

# 4. 사용 시나리오 (Use Case)

### 사용 흐름

| 단계 | 사용자 행동 | 시스템 동작 |
| --- | --- | --- |
| 1 | 가족이 웹에서 어르신 등록 | AI가 어르신 선호 호칭·건강 정보를 입력받음 |
| 2 | 문자 수신 | AI가 오전 문자 전송 (예: “좋은 아침이에요! 오늘 날씨 좋네요 ☀️”)
시간대별 자동화 대화 시작 |
| 3 | 어르신 문자 답장 (예: “잘 잤어요”) | 감정 분석 후 긍정 반응으로 공감 답변 + 건강 질문 (예: “다행이네요! 아침약 드셨어요?”) |
| 4 | 어르신 답장 (예: “약 아직...”) | 리마인더 강화 + 가족에게 미복용 알림 |
| 5 | 24시간 응답 없음 | 가족 대시보드에 ‘안부확인 필요’ 알림(푸시/웹 알림) |

# 5. 기능 정의 (Feature List)

| 기능 이름 | 기능 설명 | 중요도 |
| --- | --- | --- |
| AI 문자 대화 | 가족처럼 자연스러운 대화, 감정 인식·맥락 이해 기반 응답 | ★★★★★ |
| 건강 리마인더 | 약 복용 시간, 혈압 측정, 식사 확인 등 자동 리마인드 | ★★★★★ |

| 가족 대시보드 | 대화 로그, 감정 히트맵, 건강 트렌드(리마인더 이행률 등) 시각화 | ★★★★☆ |
